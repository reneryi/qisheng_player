## 说明

本版本为预发布版本，重点用于验证 1.7.0 的 UI 体验重塑、播放详情页稳定性和 Windows 发布打包链路。

## 新增

- 新增 UI 视觉风格模式：玻璃拟态 / 高对比极简，并在设置页提供切换入口。
- 新增 UI 效果强度与 Windows 背景材质设置，支持平衡、视觉、性能和 Auto、Mica、Acrylic、关闭等模式。
- 新增共享主框架、全局底部播放器、`Cp*` 兼容组件层和 shadcn 主题桥，统一页面表面、按钮、动效和底部播放控制。
- 新增 Now Playing 内容视图拆分、媒体信息面板和幕后信息展示，支持读取并展示 `composer`、`arranger`。
- 新增播放会话恢复，记录上次歌曲、队列、索引和播放进度。
- 新增播放、歌词、桌面歌词控制抽象，便于 UI 解耦和测试注入。

## 改进

- 主界面改为浮层侧栏、页面面板和底部浮动播放器布局，大屏侧边栏支持展开 / 收起。
- 页面标题区、设置页、文件夹页、音乐 / 专辑 / 艺术家列表、播放队列和 Now Playing 页面统一到新的视觉节奏。
- 搜索入口移入页面头部，音乐页支持展开式内联搜索。
- 动态主题更克制地影响强调色和背景渐变，降低封面取色对整体层级的干扰。
- 主路由转场改为 shared-axis 风格，底部播放器封面与 Now Playing 封面接入共享 Hero 转场。
- Windows 打包流程迁移到 `dist/windows/`，整合主程序和桌面歌词子程序，并统一产物命名。
- 项目结构文档、发布流程文档、发布说明和 README 同步新的目录职责。

## 修复

- 修复最大化后切换全屏时底部播放器 Slider 收到 0 宽约束导致 Flutter 断言并卡死的问题。
- 修复底部播放器进入 Now Playing 时重复 Hero tag 导致卡死的问题。
- 修复底部播放器在测试或独立挂载场景下读取路由可能抛 `GoError` 的问题。
- 清理代码和文档中的中文 mojibake 乱码，并恢复播放器、队列、桌面歌词等可见中文文案。
- 修复系统主题读取失败时缺少回退值的问题。
- 修复旧偏好可能导致启动页、音乐页默认排序、启动音量和静音快捷键异常的问题。
